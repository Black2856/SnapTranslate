# SnapTranslate 要件定義

## 1. 目的

SnapTranslate は、Windows 上で利用する個人向けの翻訳支援アプリケーションである。
画面上の指定範囲を OCR して LLM で翻訳し、対象位置にオーバーレイ表示する機能と、任意入力したテキストを LLM で翻訳してクリップボードへ自動コピーする機能を提供する。

## 2. 対象環境

- OS: Windows
- 実装言語: Python
- パッケージ管理・実行環境: uv
- GUI/Windows 制御: pywin32 を中心に利用する
- LLM: ChatGPT API
- 利用形態: 個人利用

## 3. 基本方針

- オブジェクト指向で設計し、機能ごとの責務を分離する。
- メインルート直下のファイル数は最小限にし、アプリケーションコードは `src/snaptranslate/` 配下に配置する。
- 設定値は GUI から変更できるようにする。
- セキュリティ対策は個人利用に必要な最小限とする。ただし API キーの平文ログ出力や例外表示は避ける。
- OCR、翻訳、オーバーレイ、入力欄、ホットキー、設定管理は差し替えやすい構造にする。
- アプリはタスクトレイ常駐を基本とし、設定画面や終了操作へアクセスできるようにする。

## 4. 機能要件

### 4.1 画面範囲 OCR 翻訳機能

#### 4.1.1 翻訳開始

- ユーザーは A 指定キーを押して OCR 翻訳を開始できる。
- A 指定キーは GUI の設定画面から変更できる。
- A 指定キーはグローバルホットキーとして動作し、アプリが前面にない場合でも反応する。

#### 4.1.2 範囲指定

- OCR 対象範囲は以下の 2 方式を提供する。
  - 事前設定方式: GUI または範囲選択画面であらかじめ OCR 対象範囲を保存する。
  - 逐次指定方式: A 指定キー押下時に毎回 OCR 対象範囲をドラッグ等で指定する。
- どちらの方式を使うかは GUI の設定画面から変更できる。
- 事前設定方式では、保存済み範囲が未設定の場合、範囲設定を促す。

#### 4.1.3 OCR

- 指定範囲のスクリーンショットを取得し、OCR によりテキストを抽出する。
- OCR の結果が空、または翻訳に不十分な場合は、翻訳 API を呼び出さずエラー状態を表示する。
- OCR エンジンは PaddleOCR を採用する。ただし将来差し替え可能な構造とする。

#### 4.1.4 LLM 翻訳

- OCR で得たテキストを ChatGPT API に送信し、翻訳結果を取得する。
- 翻訳プロンプト、翻訳方針、モデル名は設定値として管理する。
- 翻訳先言語と原文言語の扱いはユーザー設定のプロンプトに含め、原文言語の判定は LLM に任せる。
- 翻訳中はステータス表示で読み取り側が処理中であることを示す。
- API エラー、通信エラー、レート制限、タイムアウト時は、オーバーレイまたはステータス表示でユーザーに分かる形で通知する。

#### 4.1.5 オーバーレイ表示

- 翻訳結果は透明オーバーレイ上に表示する。
- オーバーレイは pywin32 を用いて実装する。
- オーバーレイ表示位置は OCR 指定範囲の位置に合わせる。
- オーバーレイの背景ボックスは透明度 0% とし、翻訳テキストのみ視認できる状態を基本とする。
- 翻訳テキストの色、フォント、サイズは GUI から設定できるようにする。
- オーバーレイ表示中に A 指定キーを押すと、表示中のオーバーレイを非表示にする。
- オーバーレイの表示状態はステータス表示に反映する。

### 4.2 テキスト入力翻訳・自動コピー機能

#### 4.2.1 入力欄表示

- ユーザーは B 指定キーでテキスト入力欄の表示/非表示を切り替えられる。
- B 指定キーは GUI の設定画面から変更できる。
- B 指定キーはグローバルホットキーとして動作し、アプリが前面にない場合でも反応する。

#### 4.2.2 テキスト入力

- 入力欄に翻訳したいテキストを入力できる。
- Enter キー押下で翻訳を実行する。
- Shift+Enter で改行できるようにする。
- 翻訳実行中は、入力側が処理中であることをステータス表示に反映する。
- 入力欄の非表示時に未送信テキストを保持するか破棄するかは設定可能にする。

#### 4.2.3 翻訳とコピー

- 入力されたテキストを ChatGPT API に送信し、翻訳結果を取得する。
- 翻訳結果は自動的にクリップボードへコピーする。
- 翻訳結果は入力欄には表示せず、コピー完了状態のみ示す。
- コピー完了時はステータス表示、入力欄、または通知で完了を示す。
- 翻訳に失敗した場合はクリップボードを更新せず、エラー状態を表示する。

### 4.3 ステータス表示

- 画面右上など、作業を邪魔しにくい位置に小さなステータス表示を出す。
- 表示例:
  - `[入力]: 翻訳中`
  - `[読み取り]: 表示中`
  - `[読み取り]: 翻訳中`
  - `[入力]: コピー完了`
- ステータス表示は GUI の設定画面から非表示にできる。
- ステータス表示は常に最前面に表示する。
- 長時間残り続ける必要がない一時状態は、一定時間後に通常表示へ戻す。

### 4.4 設定 GUI

- 以下の設定を GUI から変更できる。
  - A 指定キー
  - B 指定キー
  - OCR 範囲指定方式
  - 事前設定 OCR 範囲
  - ステータス表示の有効/無効
  - 翻訳プロンプト
  - オーバーレイ文字色、フォント、サイズ
  - ChatGPT API モデル
  - API キーの参照方法
  - 入力欄非表示時の未送信テキスト保持方針
- 設定変更後、必要に応じてホットキー登録を更新する。
- 不正なホットキーや重複したホットキーは保存前に検出する。

## 5. 非機能要件

### 5.1 操作性

- ホットキー操作を中心に、翻訳開始から表示またはコピーまでの手数を少なくする。
- オーバーレイやステータス表示は作業画面を大きく隠さない。
- エラー時はユーザーが原因を推測できる短いメッセージを表示する。

### 5.2 性能

- OCR と翻訳 API 呼び出しは UI スレッドをブロックしない。
- 連続実行時に多重リクエストが発生しないよう、同一機能内では処理中の再実行を制御する。
- API タイムアウト時間を設定値として持つ。

### 5.3 保守性

- OCR エンジン、LLM クライアント、オーバーレイ表示、ホットキー制御を独立したクラスとして分離する。
- 設定値は型付きの設定モデルで扱う。
- 例外処理とログ出力の方針を共通化する。

### 5.4 セキュリティ

- ChatGPT API キーをログに出力しない。
- API キーは環境変数またはローカル設定ファイルから読み込む。
- ローカル設定ファイルを利用する場合は、Git 管理対象外にする。
- PaddleOCR のモデルやキャッシュは Git 管理対象外にする。
- 個人利用を前提とし、ユーザー認証や複数ユーザー権限管理は対象外とする。

## 6. 想定しない範囲

- 複数ユーザー利用
- サーバー化、Web API 化
- 翻訳履歴のクラウド同期
- 必須機能としての翻訳履歴保存
- 厳格な秘密情報管理
- macOS/Linux 対応
- オフライン LLM 翻訳
