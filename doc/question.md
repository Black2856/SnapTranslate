# SnapTranslate 質問事項

## 回答済み・設計反映済み

1. PaddleOCR のモデル取得は初回起動時の自動ダウンロードでよいか。
   - 代替案: インストール手順で事前にモデルを準備する。
   - 回答: 初回起動時の自動ダウンロードでよい。モデル類は Git commit に含まれないようにする。

2. 翻訳プロンプトは読み取り翻訳と入力翻訳で別々に設定できるようにするか。
   - 現在の設計案: `read_translation_prompt` と `input_translation_prompt` を分ける。
   - 回答: 別々に設定できる設計でよい。

3. 翻訳履歴を実装する場合、保存対象は入力文と翻訳結果の両方でよいか。
   - 回答: 入力文と翻訳結果の両方を保存してよい。

4. 翻訳履歴を実装する場合、保存形式はローカル JSONL でよいか。
   - 回答: ローカル JSONL でよい。

## 以前の回答済み事項

- OCR エンジンは PaddleOCR を採用する。
- 翻訳先言語、原文言語、翻訳方針はユーザー編集可能な翻訳プロンプトで制御する。
- 原文言語の判定は LLM に任せる。
- オーバーレイの翻訳テキスト色、フォント、サイズは設定可能にする。
- 逐次範囲指定はドラッグ選択で行う。
- 入力欄は Shift+Enter で改行できるようにする。
- 入力翻訳の結果は入力欄へ表示せず、クリップボードコピーのみ行う。
- アプリはタスクトレイ常駐とする。
- ChatGPT API モデル初期値は `gpt-5.4-mini` を候補とする。
- 翻訳履歴保存は必須ではなく、工数が少ない場合のみ実装する。
